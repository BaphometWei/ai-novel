### Delegated subagents

`Intake hard gate` 只适用于直接接收 operator 新任务、review request 或 ad-hoc start 的顶层代理。

subagent 是否属于被委派任务，不能依靠自我推断，必须由主代理在委派说明中明确标记。主代理分发 subagent 任务时，必须包含一个 delegation marker，例如 `Delegated subagent task`，并写清任务范围、允许的本地动作和预期输出。具体措辞可以变化，但“这是主代理委派的 subagent 任务”这一点必须显式出现。

收到明确 delegation marker 的 subagent 不执行 `Intake hard gate`，也不把主代理的委派说明视为新的 operator intake。它不需要等待 operator-style “继续”口令才能阅读文档、检查代码、运行允许的命令、写测试、实现或完成被分配的任务。

### Intake hard gate

```text
回答任何任务前，按以下顺序进行：

对齐
- 用一句话说清你当前对任务的理解。用对方的语言，不堆术语。
- 只要有任何怀疑，先说怀疑并停下。
- 主动点出：你这次最可能猜错在哪、你在哪里替用户做了他可能
  并不知道的选择。
- 未得到“继续”前，不得查文档、总结、实现。
- 用户不需要给出"正确答案"——他只需要能说"错了"或"继续"。

表态
- 如果你对"这个任务按现在这样做下去是否合适"有任何形式、
  任何大小的怀疑——把它完整、原样说在最前面。
- 不要压缩，不要归类，不要替用户决定哪个值得说。
  不要藏进结尾的"其他考虑"。
- 说完停下，等用户明确说"继续"之后再开始具体工作。

交付方式
- 工作中途冒出新的怀疑——停下，回到"表态"，不要带着怀疑
  把工作做完。
```
